Tahap 1

Data Wrangling (Rivaldo)

Apa itu Data Wrangling?

Data wrangling adalah proses membersihkan dan menyiapkan data mentah agar siap dianalisis. Data dari internet seringkali kotor — ada kolom tidak perlu, tipe data salah, nilai kosong, atau duplikat. Tahap ini memastikan data yang masuk ke analisis sudah bersih dan konsisten.

Dataset yang Dipakai

  • Sumber: Kaggle — "Bakery Sales Data 2006–19"
  • Jumlah baris: 5.113 baris (jauh di atas minimum 1.000)
  • Kolom awal: 9 kolom (Unnamed:0, Date, weekday, cakes, pies, cookies, smoothies, coffee, promotion)
  • Isi data: Penjualan harian 5 produk bakery selama 14 tahun

Langkah-langkah Cleaning

1. Download Dataset

Menggunakan library opendatasets untuk download langsung dari Kaggle ke Colab.

import opendatasets as od
od.download("https://www.kaggle.com/datasets/sanu12300/bakery-sales-data-2006-19")

2. Cek Struktur Data

df.info() menampilkan tipe data tiap kolom dan jumlah non-null. df.shape menampilkan (jumlah baris, jumlah kolom).

df.info()        # lihat tipe data & non-null count
df.head()        # lihat 5 baris pertama
print(df.shape)  # output: (5113, 9)

3. Cek Missing Value & Duplikat

Hasilnya: 0 missing value, 0 duplikat — data sudah bersih dari sisi ini.

df.isnull().sum()    # cek nilai kosong per kolom
df.duplicated().sum()  # cek baris duplikat → hasilnya 0

4. Konversi Tipe Data Date

Kolom Date awalnya bertipe object (string). Diubah ke datetime64 agar bisa digunakan untuk operasi tanggal (bulan, tahun, minggu).

df['Date'] = pd.to_datetime(df['Date'])
# Setelah ini bisa pakai: df['Date'].dt.month, .dt.year, dll.

5. Encoding Kolom Promotion

Kolom promotion berisi teks "promotion" / "none". Diubah ke angka 1 / 0 agar bisa diproses secara numerik (korelasi, rata-rata, dsb).

df['promotion'] = df['promotion'].map({
    'none': 0,
    'promotion': 1
})
Kenapa harus di-encode? Komputer tidak bisa menghitung rata-rata dari string "promotion". Setelah jadi angka (0/1), kita bisa langsung pakai df.groupby('promotion').mean() untuk membandingkan penjualan saat promo vs tidak.

6. Hapus Kolom Tidak Perlu & Tambah Kolom Waktu

df = df.drop(columns=['Unnamed: 0'])   # kolom index duplikat, tidak perlu
df['month'] = df['Date'].dt.month       # ekstrak bulan untuk EDA
df['year']  = df['Date'].dt.year        # ekstrak tahun untuk tren tahunan
✅ Hasil Akhir Wrangling: Dataset bersih dengan 5.113 baris, 8 kolom, tanpa missing value, tanpa duplikat. Tipe data sudah benar semua, kolom promotion sudah numerik, dan sudah ada kolom tambahan month & year untuk keperluan EDA.

Exploratory Data Analysis — EDA (Argya)

Apa itu EDA?

EDA adalah proses menjelajahi data untuk menemukan pola, tren, dan anomali sebelum membuat kesimpulan. Di sini kita belum menjawab pertanyaan spesifik — kita dulu bertanya "ada apa di data ini?"

Temuan Utama EDA

59.99
rata2 cookies/hari
49.84
rata2 coffee/hari
25.03
rata2 cakes/hari (terendah)
  • Produk terlaris: Cookies (306.719 unit total) — 2.4x lebih banyak dari Cakes (127.957 unit)
  • Hari paling ramai: Jumat (207.4 unit/hari), paling sepi Selasa (199.6 unit/hari) — selisihnya tidak drastis, penjualan stabil sepanjang minggu
  • Promosi paling efektif pada Cookies: naik +91.8% saat ada promo
  • Korelasi antar produk mendekati 0: artinya pelanggan membeli satu produk tidak berarti membeli produk lain di hari yang sama (tidak ada cross-selling organik)

Syntax Penting EDA

Membuat kolom total & kolom waktu tambahan:

df['total'] = df[['cakes','pies','cookies','smoothies','coffee']].sum(axis=1)
# axis=1 artinya jumlahkan per BARIS (bukan per kolom)

df['month_name'] = df['Date'].dt.strftime('%b')   # → "Jan", "Feb", dst
df['week'] = df['Date'].dt.isocalendar().week.astype(int)

Analisis promosi — membandingkan rata-rata saat promo vs tidak:

# groupby('promotion') → bagi data jadi 2 grup: 0 (tidak promo) dan 1 (promo)
# .mean() → hitung rata-rata tiap produk di tiap grup
# .T → transpose agar produk jadi baris, bukan kolom
promo_compare = df.groupby('promotion')[products].mean().T
promo_compare.columns = ['Tanpa Promo', 'Ada Promo']

# Hitung % kenaikan
promo_compare['kenaikan_%'] = (
    (promo_compare['Ada Promo'] - promo_compare['Tanpa Promo'])
    / promo_compare['Tanpa Promo'] * 100
).round(2)

Analisis korelasi:

corr_matrix = df[products + ['promotion']].corr()
# .corr() menghitung koefisien korelasi Pearson antar semua pasangan kolom
# Nilai -1 s/d +1: mendekati 0 = tidak ada hubungan linear

Tahap 4

Analisis & Identifikasi Masalah (Rivaldo)

3 Masalah yang Ditemukan

Masalah 1 — Ketimpangan Penjualan Produk

Cookies (306K unit) vs Cakes (127K unit) → selisih 2.4x. Dua produk (cookies + coffee) sudah menyumbang 54% total penjualan. Cakes hanya 12.3%. Apakah sumber daya bakery sudah dialokasikan optimal?

Masalah 2 — Efektivitas Promosi Tidak Merata

Cookies naik +91.8% saat promo, tapi Cakes hanya +26.1%. Jika promosi dialokasikan tanpa data, bisa jadi kita promo produk yang kurang responsif dan membuang-buang anggaran.

Masalah 3 — Tidak Ada Cross-Selling Organik

Korelasi antar produk ≈ 0 berarti pelanggan beli cookies tidak berarti beli coffee di hari yang sama. Ini peluang: bundling produk bisa mendorong penjualan produk lemah.

Rumusan Masalah Utama SPK:
"Produk mana yang harus diprioritaskan untuk mendapat alokasi promosi pada periode berikutnya?"

Implementasi SPK — SAW & TOPSIS (Rivaldo)

Apa itu SPK?

SPK (Sistem Pendukung Keputusan) adalah sistem yang membantu pengambil keputusan memilih dari beberapa alternatif berdasarkan banyak kriteria secara terstruktur dan objektif. Di proyek ini: memilih produk mana yang paling layak diprioritaskan untuk promosi, berdasarkan 4 kriteria dari data historis.

Matriks Keputusan (Langkah Awal Semua SPK)

Matriks keputusan adalah tabel utama yang berisi nilai tiap alternatif (produk) untuk setiap kriteria. Ini adalah input dari semua metode SPK.
